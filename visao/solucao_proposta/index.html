    Solução Proposta¶

    A proposta do ClassHub é oferecer uma plataforma web para apoiar o controle de presença em aulas práticas e laboratoriais. A solução leva o processo para um ambiente digital no qual professores, administradores e alunos possuem fluxos compatíveis com suas responsabilidades.

    O foco da solução não é apenas criar uma tela de chamada. O sistema deve ajudar a organizar o fluxo completo da frequência: cadastro de turmas, controle dos alunos, registro das aulas, acompanhamento de presenças e geração de relatórios.

    Visão geral da solução¶

    A plataforma é um sistema de gestão de presença acadêmica acessado por usuários autenticados. Por meio dela, é possível acompanhar dados das turmas, registrar presenças e faltas durante as aulas e consultar históricos de frequência.

    Com isso, a instituição passa a ter uma base mais centralizada e confiável, reduzindo a dependência de listas impressas e conferências manuais.

    Principais funcionalidades¶

    Para atender ao problema apresentado, a solução deve contemplar os seguintes pontos:

    • Dashboard com informações sobre alunos, aulas cadastradas, taxa média de presença, alertas de baixa frequência e gráficos administrativos.
    • Gerenciamento administrativo de turmas, permitindo criar, editar, remover e definir o professor responsável.
    • Cadastro e visualização de estudantes vinculados às turmas.
    • Registro digital de presença e falta durante as aulas.
    • Registro da data e horário das aulas.
    • Consulta ao histórico de frequência dos alunos.
    • Geração de relatórios de presença individual, percentual de presença, alunos com frequência inferior a 75% e histórico por turma.
    • Autenticação para proteger o acesso a dados sensíveis.
    • Interface responsiva para uso em desktop, tablet e celular.

    Como a solução ajuda no problema¶

    Com o registro digital, o professor deixa de depender de listas impressas para controlar a frequência. As informações passam a ser registradas diretamente na plataforma, diminuindo a chance de perda de dados, rasuras e erros na transcrição posterior.

    Para a coordenação e a equipe administrativa, a centralização dos dados facilita o acompanhamento das turmas e a geração de relatórios. Em vez de buscar informações em papéis ou planilhas separadas, o sistema passa a concentrar os dados em um único lugar.

    Outro ponto importante é a possibilidade de acompanhar alunos com baixa frequência com mais rapidez. Isso permite que a instituição identifique situações de atenção antes que o problema cresça.

    Tecnologias e ferramentas utilizadas¶

    Para construir a solução, a equipe definiu um conjunto de tecnologias e ferramentas que apoiam o desenvolvimento, a documentação, a comunicação e a publicação do projeto.

    Área Tecnologia/Ferramenta Uso no projeto
    Front-end Next.js Desenvolvimento da interface web da aplicação.
    Front-end HTML Estruturação semântica das páginas e componentes.
    Front-end Tailwind CSS Estilização da interface e construção de telas responsivas.
    Front-end JavaScript Implementação da lógica da interface e integração com o back-end.
    Back-end NestJS Desenvolvimento da API e organização das regras do servidor.
    Back-end API REST Comunicação entre front-end e back-end por meio de endpoints.
    Banco de dados SQLite Persistência de dados relacionados a usuários, turmas, aulas e frequência.
    Versionamento Git Controle de versão das alterações realizadas no projeto.
    Repositório GitHub Armazenamento do código, colaboração da equipe e organização das entregas.
    Documentação MkDocs Construção e organização da documentação do projeto.
    Publicação da documentação GitHub Pages Disponibilização da documentação online.
    Deploy do front-end Vercel Publicação da interface web para apresentação e validação.
    Execução do back-end Ambiente Node.js com armazenamento persistente Execução da API NestJS e preservação do arquivo SQLite.
    Prototipação Figma Criação de protótipos e alinhamento visual das telas.
    Comunicação Discord Alinhamentos, avisos, dúvidas e reuniões da equipe.

    Essa combinação de ferramentas foi escolhida para apoiar tanto a parte técnica quanto a organização do trabalho em equipe. Assim, o projeto consegue manter uma base de código versionada, uma documentação acessível e uma comunicação mais clara entre os integrantes.

    Resultado esperado¶

    Espera-se que a solução torne o controle de presença mais prático, confiável e fácil de acompanhar. A instituição deve conseguir reduzir o uso de papel, diminuir retrabalho, evitar parte dos erros manuais e ter mais clareza sobre a frequência dos alunos nas aulas práticas e laboratoriais.

    No fim, a plataforma deve servir como uma ferramenta de apoio para professores, coordenação e equipe administrativa, melhorando a rotina de controle acadêmico sem tornar o processo mais complicado para quem vai usar o sistema.
